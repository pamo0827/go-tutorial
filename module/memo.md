# Goチュートリアルで学習したコマンドのまとめ

このチュートリアルでは、Go言語のツールチェーンを構成する様々なコマンドを使用してきた。各コマンドの役割と使用法は以下の通りである。

### 1. プロジェクトと依存関係の管理

Goのプロジェクトは「モジュール」として管理され、以下のコマンドで操作する。

- **`go mod init <モジュール名>`**
    - **役割**: 新しいモジュールを初期化し、依存関係を管理するための `go.mod` ファイルを作成する。
    - **使用例**: `go mod init example.com/greetings`
- **`go mod tidy`**
    - **役割**: ソースコードの `import` 文を解析し、`go.mod` ファイルに必要な依存関係を追加・不要なものを削除する。
- **`go mod edit -replace=<モジュール>=<ローカルパス>`**
    - **役割**: モジュールのインポートパスを、ローカルファイルシステム上の別のパスに置き換える。未公開のローカルモジュールをテストする際に使用する。
    - **使用例**: `go mod edit -replace=example.com/greetings=../greetings`

### 2. コードのビルド、実行、インストール

コードをコンパイルし、実行するためのコマンド群。

- **`go run .`**
    - **役割**: 現在のディレクトリにあるメインパッケージをコンパイルし、直接実行する。一時的な実行可能ファイルを生成するが、保存はされない。開発中の素早い動作確認に便利である。
    - **使用例**: `go run .`
- **`go build`**
    - **役割**: 現在のディレクトリにあるパッケージをコンパイルし、実行可能ファイル（Windowsでは `.exe`）をその場に生成する。`go run` とは異なり、実行ファイルが保存される。
    - **使用例**: `go build`
- **`go install`**
    - **役割**: パッケージをコンパイルし、Goの `bin` ディレクトリ（通常は `%USERPROFILE%\go\bin`）に実行可能ファイルをインストールする。`PATH` 環境変数が正しく設定されていれば、どのディレクトリからでもコマンド名だけで実行できるようになる。
    - **使用例**: `go install ./cmd/tour`

### 3. テスト

コードの品質を保証するためのテストを実行する。

- **`go test`**
    - **役割**: 現在のディレクトリにあるテストファイル（`_test.go` で終わるファイル）内のテスト関数を実行する。
    - **使用例**: `go test`
- **`go test -v`**
    - **役割**: テストを詳細（verbose）モードで実行し、各テストの成功・失敗を個別に表示する。
    - **使用例**: `go test -v`

### 4. 環境と情報の確認

Goの環境設定を確認・変更するためのコマンド。

- **`go list -f '{{.Target}}'`**
    - **役割**: `go install` が実行可能ファイルをどこにインストールするか、そのフルパスを表示する。
    - **使用例**: `go list -f '{{.Target}}'`
- **`go env -w GOBIN=<パス>`**
    - **役割**: `go install` のインストール先を、デフォルトの場所から指定したパスに変更する。通常は設定不要である。
    - **使用例**: `go env -w GOBIN=C:\my-go-bin`

### 5. 関連する一般コマンド

チュートリアルを進める上で使用した、Go以外のコマンド。

- **`git clone <URL>`**
    - **役割**: Gitリポジトリからソースコードをローカルにダウンロードする。
    - **使用例**: `git clone https://github.com/atotto/go-tour-jp.git`
- **`echo $env:PATH` (PowerShell)**
    - **役割**: システムの `PATH` 環境変数の内容を表示する。Goの `bin` ディレクトリが含まれているか確認する際に使用した。
